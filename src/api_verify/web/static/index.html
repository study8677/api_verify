Is your API relay actually serving the model it claims?

Paste a relay endpoint, an API key, and the model name. We run 5 capability and protocol probes designed to expose cheap-model substitution, parameter drops, and capability degradation common to premium models like gpt-4.1, claude-opus-4-x, and gemini-2.5-pro.

Result is an evidence-based risk score, not cryptographic proof. model/usage/system_fingerprint can all be forged.

How the probes work

  • Metadata echo — Does the response shape and returned model field line up with what the request asked for?
  • JSON mode — Tells if the relay honors response_format: json_object or silently drops it.
  • Strict tool calling — Tests whether the relay preserves OpenAI-style strict tool schemas. Cheap models often degrade tool calls to plain text.
  • Error semantics — Sends an intentionally invalid temperature=99. Real upstreams return a 4xx that names the offending parameter. Relays that silently clamp it are suspicious.
  • Long-context needle — Plants a passcode inside ~12K characters of padding and asks for it back. Substituted cheap models fail here.

Deep mode additionally runs a deterministic-arithmetic reasoning probe and a knowledge-cutoff probe.

Privacy & safety

  • API keys are used once per request and dropped from memory after the verification. They are not written to disk, logs, or any storage.
  • We do not log request or response bodies. Only protocol, model, IP-prefix, and timing.
  • Rate-limited to 5 verifications per hour per IP to limit cost & abuse.
  • For your own safety: prefer a short-lived / low-budget key when testing untrusted relays.
